# Contributing

Contributions are welcome! All pull requests require approval from the maintainer or a designated contributor before being merged.

## How to contribute

1. **Open an Issue** first to describe the feature or bug fix you have in mind
2. **Fork this repo** and make your changes on a feature branch
3. **Open a Pull Request** — include a clear description of what changed and why
4. **Wait for review** — a maintainer or contributor will review and either approve, request changes, or close the PR

## Guidelines

- Keep changes focused — one feature or fix per PR
- Follow the existing code style (TypeScript, no unnecessary comments)
- Test your changes in Chrome before submitting
